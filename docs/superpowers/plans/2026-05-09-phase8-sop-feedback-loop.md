# Phase 8 自动 SOP 生成 + 反馈闭环 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 让 AIOps 不只是"记得"过去案例（Phase 7 已做），而是**自动总结成可复用 SOP** + **从人工反馈中学习避坑**。每次 workflow 完成后，异步生成飞书故障简报；累积 N 次同类成功处置后，LLM 自动产出 SOP markdown 候选，提 git PR 让运维 review；运维拒绝/修改 agent 决策的行为反向标注成"反例"，下次注入到 prompt 里避坑。

**Architecture:** 三个独立子系统组合：
1. **故障简报**（每次 workflow 完成后立刻发）：固定模板 + 一段 LLM 生成的人话总结，推飞书运维群
2. **SOP 候选生成器**（定时任务）：每天扫一次 PG，找出"同 host + 同 runbook 且 ≥ 3 次成功"的告警类型，调 LLM 按 [Hermes SKILL.md 结构](https://medium.com/@rosgluk/hermes-agent-skill-authoring-skill-md-structure-and-best-practices-32a59d09189b) 生成候选，写到 `docs/sop/<category>/<name>.md`，开一个 git PR
3. **反例标注**（基于 Phase 7 表，加新字段 + 新查询逻辑）：人工拒绝 / 修改 agent plan 的行为标注成 "feedback"，下次 query_similar_cases 时把反例也注入

**Tech Stack:** Python 3.14 + asyncpg（已有）+ pyyaml（已有）+ git CLI（在容器里调 subprocess）+ apscheduler（新增，最轻量定时器）

## 设计权衡

### 为什么 SOP 候选要走 git PR 而不是直接落盘

LLM 生成的 SOP **必须人工 review** 才能用——参考架构文档 §9.3："Runbook 不是普通脚本，是直接操作生产的代码，必须严格管理"。SOP 是 Runbook 的"使用说明书"，同样要 git tracked + PR review。

实现路径：
- 容器里跑 `git config user.name aiops-bot` + `git checkout -b sop/<auto-generated>`
- 写 markdown 到 `docs/sop/<category>/<name>.md`
- `git push origin <branch>` + `gh pr create --title "auto-SOP: ..."`
- 容器需要 GitHub token（写入 `.env` 的 `GITHUB_TOKEN`）

### 为什么不直接 train fine-tune

候选思路："反例" + "成功案例" 直接喂 SFT。**reject**，理由：
- 内网数据量太小，几十条 case 够不上 fine-tune 阈值
- 维护 fine-tuned model 成本高（推理基建 / 评估 / 回滚）
- 直接把反例**注入 prompt** 已经能解决 80% 问题（in-context learning）

参考 python-performance-optimization 原则：**先用最简单方案达到目标，再考虑优化**。

### Hermes SKILL.md 结构借鉴

按 Hermes Agent 的 SKILL.md 模板（Medium 文章总结的 5 段式）：

```yaml
---
name: redis-restart-on-oom
description: 检测到 redis OOM 类告警时，按以下步骤自动处置
version: 1.0
tags: [redis, oom, restart]
---

## When to use
- 触发条件 1
- 触发条件 2

## Quick reference
- 关键命令 / 路径

## Procedure
1. step 1
2. step 2
...

## Pitfalls
- 已知失败案例

## Verification
- 成功标准
```

**progressive disclosure**：飞书简报 + agent prompt 只看 `description`（轻量）；运维真要看详情时手动打开 markdown。

### 性能预算

故障简报（实时）：
- 每个 workflow 完成后 1 次 LLM 调用 → 加 ~3s 延迟（用户感知后台）
- 用 fire-and-forget 异步任务，不阻塞 workflow.return

SOP 候选生成（定时）：
- 每天凌晨 3 点跑 1 次，单次预算 < 5 分钟
- 单天最多生成 5 个候选 PR（防止刷屏）

反例查询（每次 RCA 前）：
- 复用 Phase 7 的 GIN 索引，加 `feedback_label` 列 partial index
- 单次查询 < 50ms

```sql
-- Phase 8: 加反馈标注列
ALTER TABLE audit_records
    ADD COLUMN IF NOT EXISTS feedback_label TEXT,           -- 'rejected_correctly' / 'rejected_wrongly' / 'modified' / null
    ADD COLUMN IF NOT EXISTS feedback_reason TEXT,          -- 人工填的原因（拒绝时表单）
    ADD COLUMN IF NOT EXISTS feedback_at TIMESTAMPTZ;

-- 反例查询专用 partial index（只索引被标注的少数行）
CREATE INDEX IF NOT EXISTS idx_audit_feedback_label
    ON audit_records (feedback_label, host_ip)
    WHERE feedback_label IS NOT NULL;

-- SOP 候选源数据视图：累积 ≥3 次成功的 (host_ip, runbook_id, event_name 关键词) 组合
CREATE OR REPLACE VIEW sop_candidates AS
SELECT
    host_ip,
    runbook_id,
    -- 按 event_name 第一个词分组（粗粒度去重）
    split_part(event_name, ' ', 1) AS event_keyword,
    count(*) AS success_count,
    array_agg(id ORDER BY created_at DESC) AS sample_ids,
    max(created_at) AS last_success_at
FROM audit_records
WHERE verify = true
  AND decision IN ('approved', 'auto_approved')
  AND runbook_id IS NOT NULL
GROUP BY host_ip, runbook_id, split_part(event_name, ' ', 1)
HAVING count(*) >= 3;
```

`src/hermes/feedback.py`:

```python
"""反馈数据访问层

性能要点：
- find_negative_cases 用 partial index（只索引带 feedback 的行），
  比扫全表快 N 倍（N = 总告警数 / 反例数）
- 单次只取 LIMIT 3（注入 prompt 用，不需要更多）
"""

from datetime import datetime
from enum import Enum

import asyncpg

from src.hermes.models import AuditRecordRead


class FeedbackLabel(str, Enum):
    REJECTED_WRONGLY = "rejected_wrongly"   # 运维拒绝，agent 提议有问题
    REJECTED_CORRECTLY = "rejected_correctly"  # agent 提议本来对的，但场景不该自动处理
    MODIFIED = "modified"                    # 运维修改了 plan 后才执行（暂不实现，未来扩展）


class FeedbackRepository:
    def __init__(self, pool: asyncpg.Pool, schema: str = "public") -> None:
        self.pool = pool
        self.schema = schema

    @property
    def _table(self) -> str:
        return f"{self.schema}.audit_records"

    async def label(
        self, record_id: int, label: FeedbackLabel, reason: str = ""
    ) -> None:
        async with self.pool.acquire() as conn:
            await conn.execute(
                f"""
                UPDATE {self._table}
                SET feedback_label = $2,
                    feedback_reason = $3,
                    feedback_at = NOW()
                WHERE id = $1
                """,
                record_id, label.value, reason[:500],
            )

    async def get_with_feedback(self, record_id: int) -> AuditRecordRead | None:
        from src.hermes.repository import _row_to_read
        async with self.pool.acquire() as conn:
            row = await conn.fetchrow(
                f"SELECT *, NULL::real AS rank FROM {self._table} WHERE id = $1",
                record_id,
            )
        if not row:
            return None
        # 直接附加 feedback 字段到 read model
        rec = _row_to_read(row)
        # AuditRecordRead 没有 feedback 字段，扩展用 setattr 简化
        rec.__dict__["feedback_label"] = row["feedback_label"]
        rec.__dict__["feedback_reason"] = row["feedback_reason"]
        rec.__dict__["feedback_at"] = row["feedback_at"]
        return rec

    async def find_negative_cases(
        self, query: str, host_ip: str | None, limit: int = 3
    ) -> list[AuditRecordRead]:
        """找反例：被人工标记过的 audit"""
        if not query.strip():
            return []
        from src.hermes.repository import _row_to_read

        ts_query = " | ".join(query.split())
        host_boost = "CASE WHEN host_ip = $2 THEN 1 ELSE 0 END" if host_ip else "0"
        params = [ts_query]
        if host_ip:
            params.append(host_ip)

        sql = f"""
            SELECT *, ts_rank(fts, to_tsquery('simple', $1)) AS rank
            FROM {self._table}
            WHERE feedback_label IS NOT NULL
              AND fts @@ to_tsquery('simple', $1)
            ORDER BY ({host_boost}) DESC, rank DESC, feedback_at DESC
            LIMIT {int(limit)}
        """
        async with self.pool.acquire() as conn:
            rows = await conn.fetch(sql, *params)

        result = []
        for r in rows:
            rec = _row_to_read(r)
            rec.__dict__["feedback_label"] = r["feedback_label"]
            rec.__dict__["feedback_reason"] = r["feedback_reason"]
            result.append(rec)
        return result
```

```python
async def query_negative_cases(
    fb_repo: "FeedbackRepository",
    alert: Alert,
    limit: int = 2,
) -> list[AuditRecordRead]:
    """找过去类似告警的反例（人工标记 agent 提议错的）"""
    query = f"{alert.event_name} {alert.message}"
    return await fb_repo.find_negative_cases(query=query, host_ip=alert.host_ip, limit=limit)


def format_negative_cases(cases: list[AuditRecordRead]) -> str:
    """渲染反例段，让 agent 知道避坑"""
    if not cases:
        return ""
    lines = ["**避坑案例**（运维标记 agent 之前判断错过的）："]
    for i, c in enumerate(cases, 1):
        label = getattr(c, "feedback_label", "?")
        reason = getattr(c, "feedback_reason", "") or ""
        lines.append(
            f"{i}. [{c.created_at.strftime('%Y-%m-%d')}] {c.event_name} on {c.host_ip}\n"
            f"   - agent 当时提议: runbook=`{c.runbook_id}` params={c.runbook_params}\n"
            f"   - 运维反馈: {label} - {reason[:200]}"
        )
    return "\n".join(lines)
```

```python
"""每次 workflow 完成后异步发一条飞书故障简报

性能要点：
- LLM 调用单次 ~3s，但放在 workflow 末尾的 fire-and-forget activity，不阻塞 return
- summary 控制在 200 字以内，节省 token
- 飞书发送失败不重试（Phase 8 不是关键路径）
"""

import logging

from temporalio import activity

from src.activities.feishu import _post_im_message
from src.activities.llm import llm_router

logger = logging.getLogger(__name__)


def _render_summary(
    *,
    event_id: str, decision: str, runbook_id: str | None,
    verify: bool | None, host_ip: str, event_name: str,
    agent_reasoning: str, llm_summary: str,
) -> str:
    icon = "✅" if verify else ("❌" if verify is False else "⚠️")
    return f"""**{icon} 故障简报 #{event_id}**

- **告警**: {event_name} on `{host_ip}`
- **决策**: {decision}
- **Runbook**: `{runbook_id or 'n/a'}`
- **验证**: {"通过 ✅" if verify else ("未通过 ❌" if verify is False else "n/a")}

**根因**: {agent_reasoning[:300]}

**经验沉淀**: {llm_summary}
"""


@activity.defn
async def post_incident_summary(
    event_id: str,
    decision: str,
    runbook_id: str | None,
    verify_str: str,           # "true" / "false" / "null"
    host_ip: str,
    event_name: str,
    agent_reasoning: str,
) -> None:
    """生成 + 推飞书。失败不抛错（不影响主流程）"""
    if llm_router is None:
        logger.warning("llm_router not ready, skipping summary")
        return

    verify: bool | None = None
    if verify_str == "true":
        verify = True
    elif verify_str == "false":
        verify = False

    # 让 LLM 写一句"经验沉淀"
    prompt = f"""下面是一次 AIOps 自动处置的简报。用 1~2 句话（< 80 字）总结
这次处置的经验，给运维同事看：

告警: {event_name} on {host_ip}
决策: {decision}
Runbook: {runbook_id}
验证: {verify}
Agent 推理: {agent_reasoning[:500]}

输出格式（JSON）：{{"summary": "..."}}
"""
    try:
        from pydantic import BaseModel
        class S(BaseModel): summary: str = ""
        result = await llm_router.invoke(prompt, S)
        llm_summary = result.summary
    except Exception as exc:
        logger.warning(f"summary llm failed: {exc}")
        llm_summary = "（LLM 总结失败）"

    md = _render_summary(
        event_id=event_id, decision=decision, runbook_id=runbook_id,
        verify=verify, host_ip=host_ip, event_name=event_name,
        agent_reasoning=agent_reasoning, llm_summary=llm_summary,
    )

    try:
        await _post_im_message(msg_type="text", content={"text": md})
    except Exception as exc:
        logger.warning(f"summary feishu post failed: {exc}")
```

```python
"""按 Hermes Agent SKILL.md 结构渲染 markdown

参考：https://medium.com/@rosgluk/hermes-agent-skill-authoring-skill-md-structure-and-best-practices-32a59d09189b

5 段式：
- frontmatter (YAML)
- When to use
- Quick reference
- Procedure
- Pitfalls
- Verification
"""

from dataclasses import dataclass
from datetime import datetime


@dataclass
class SkillCandidate:
    name: str          # e.g. "disk-cleanup-tmp-aiops-target"
    description: str   # 一句话
    category: str      # devops / disk / service ...
    when_to_use: list[str]
    quick_ref: list[str]
    procedure: list[str]
    pitfalls: list[str]
    verification: list[str]


def render_skill_md(candidate: SkillCandidate, sample_count: int) -> str:
    """渲染成 markdown 字符串"""
    today = datetime.utcnow().strftime("%Y-%m-%d")
    return f"""---
name: {candidate.name}
description: {candidate.description}
version: 0.1.0
author: aiops-bot
tags:
  - {candidate.category}
metadata:
  hermes:
    auto_generated: true
    generated_at: {today}
    sample_count: {sample_count}
---

## When to use

{_bullets(candidate.when_to_use)}

## Quick reference

{_bullets(candidate.quick_ref)}

## Procedure

{_numbered(candidate.procedure)}

## Pitfalls

{_bullets(candidate.pitfalls) if candidate.pitfalls else "_暂无已知坑点_"}

## Verification

{_bullets(candidate.verification)}

---

*This SOP was auto-generated by AIOps from {sample_count} historical successful cases.*
*Review before merging. Generated on {today}.*
"""


def _bullets(items: list[str]) -> str:
    return "\n".join(f"- {x}" for x in items) if items else "_n/a_"


def _numbered(items: list[str]) -> str:
    return "\n".join(f"{i+1}. {x}" for i, x in enumerate(items)) if items else "_n/a_"
```

- [ ] **Step 2: 写 sop_generator activity**

`src/activities/sop_generator.py`:

```python
"""SOP 候选生成器 - 定时任务调用

性能：
- 单次 LLM 调用平均 ~5s，每天最多 5 次（限制 candidate 数）
- 用 sop_candidates 视图聚合，PG 端做 GROUP BY
- 输出 markdown 写本地文件，git push 走 subprocess（不阻塞主进程）
"""

import logging
from typing import Any

from temporalio import activity

from src.sop.skill_template import SkillCandidate, render_skill_md
from src.activities.llm import llm_router
from src.hermes.repository import AuditRepository

logger = logging.getLogger(__name__)

_repo: AuditRepository | None = None
_MAX_CANDIDATES_PER_DAY = 5


def set_repo(repo): global _repo; _repo = repo


async def _fetch_candidates() -> list[dict[str, Any]]:
    """从 sop_candidates 视图取候选"""
    if _repo is None:
        return []
    async with _repo.pool.acquire() as conn:
        rows = await conn.fetch(
            f"SELECT host_ip, runbook_id, event_keyword, success_count, sample_ids "
            f"FROM {_repo.schema}.sop_candidates "
            f"ORDER BY success_count DESC LIMIT {_MAX_CANDIDATES_PER_DAY}"
        )
    return [dict(r) for r in rows]


async def _generate_skill_with_llm(
    host_ip: str, runbook_id: str, event_keyword: str, samples: list[dict],
) -> SkillCandidate:
    """LLM 生成 5 段式 SKILL"""
    if llm_router is None:
        raise RuntimeError("llm_router not ready")

    samples_text = "\n".join(
        f"- {s['event_name']}: {s['agent_reasoning'] or 'n/a'}"
        for s in samples[:5]
    )
    prompt = f"""根据以下 AIOps 历史成功案例，生成一个标准操作流程 (SOP) markdown，
按 Hermes SKILL.md 5 段式结构。

主机 IP: {host_ip}
告警关键词: {event_keyword}
Runbook: {runbook_id}
成功次数: {len(samples)}

历史样本：
{samples_text}

返回 JSON：
{{
  "name": "lowercase-with-hyphens",
  "description": "一句话",
  "category": "disk|service|network|etc",
  "when_to_use": ["条件 1", "条件 2"],
  "quick_ref": ["命令/路径"],
  "procedure": ["步骤 1", "步骤 2"],
  "pitfalls": ["坑点"],
  "verification": ["成功标准"]
}}
"""
    from pydantic import BaseModel
    class Out(BaseModel):
        name: str
        description: str
        category: str = "devops"
        when_to_use: list[str] = []
        quick_ref: list[str] = []
        procedure: list[str] = []
        pitfalls: list[str] = []
        verification: list[str] = []

    result = await llm_router.invoke(prompt, Out)
    return SkillCandidate(**result.model_dump())


@activity.defn
async def generate_sop_candidates() -> list[str]:
    """跑一次：找候选 → 调 LLM → 写文件。返回新生成的 markdown 文件路径列表"""
    candidates = await _fetch_candidates()
    if not candidates:
        logger.info("no SOP candidates today")
        return []

    written: list[str] = []
    for c in candidates:
        try:
            # 拉每个候选的样本详情
            sample_ids = c["sample_ids"][:5]
            async with _repo.pool.acquire() as conn:
                samples = await conn.fetch(
                    f"SELECT event_name, agent_reasoning FROM {_repo.schema}.audit_records "
                    f"WHERE id = ANY($1::bigint[])",
                    sample_ids,
                )
            samples_dicts = [dict(s) for s in samples]

            skill = await _generate_skill_with_llm(
                c["host_ip"], c["runbook_id"], c["event_keyword"], samples_dicts,
            )
            md = render_skill_md(skill, sample_count=c["success_count"])

            # 写文件
            from pathlib import Path
            sop_dir = Path("/app/docs/sop") / skill.category
            sop_dir.mkdir(parents=True, exist_ok=True)
            path = sop_dir / f"{skill.name}.md"
            path.write_text(md, encoding="utf-8")
            written.append(str(path))
            logger.info(f"sop candidate written: {path}")
        except Exception as exc:
            logger.warning(f"sop generation failed for {c}: {exc}")

    return written
```

```python
"""自动开 SOP PR

性能/可靠性：
- 全程 subprocess.run 同步调用（操作 git/gh 用 shell 最简单）
- 失败时记日志不抛错（PR 创建失败不影响主流程）
"""

import logging
import os
import subprocess
from pathlib import Path

logger = logging.getLogger(__name__)


def open_sop_pr(file_paths: list[str], branch_suffix: str) -> str | None:
    """提交 SOP markdown 文件到 git，开 PR，返回 PR URL"""
    if not file_paths:
        return None
    token = os.environ.get("GITHUB_TOKEN", "")
    if not token:
        logger.warning("GITHUB_TOKEN not set, cannot create PR")
        return None

    branch = f"sop/auto-{branch_suffix}"
    repo_dir = "/app"  # 容器里的代码挂载点
    try:
        subprocess.run(["git", "config", "--global", "user.email", "aiops-bot@local"], check=True)
        subprocess.run(["git", "config", "--global", "user.name", "aiops-bot"], check=True)
        subprocess.run(["git", "checkout", "-b", branch], cwd=repo_dir, check=True)
        subprocess.run(["git", "add", *file_paths], cwd=repo_dir, check=True)
        subprocess.run(["git", "commit", "-m", f"chore: auto-generated SOPs {branch_suffix}"], cwd=repo_dir, check=True)
        subprocess.run(["git", "push", "-u", "origin", branch], cwd=repo_dir,
                       env={**os.environ, "GIT_ASKPASS": "echo", "GH_TOKEN": token}, check=True)
        result = subprocess.run(
            ["gh", "pr", "create", "--title", f"auto-SOP {branch_suffix}",
             "--body", f"自动生成的 SOP，请运维 review:\n\n" + "\n".join(f"- {p}" for p in file_paths)],
            cwd=repo_dir, capture_output=True, text=True, env={**os.environ, "GH_TOKEN": token},
            check=True,
        )
        url = result.stdout.strip()
        logger.info(f"SOP PR opened: {url}")
        return url
    except subprocess.CalledProcessError as exc:
        logger.warning(f"git/gh failed: {exc.stderr if hasattr(exc, 'stderr') else exc}")
        return None
```
